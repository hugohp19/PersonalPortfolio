Planning the evening was created for the purpose of facilitating the user with more information in order to help them make the decision between going out or staying home and watch a movie. If the they want to “Go Out”, they only need to input the zip code or name of a city and the application will display the weather and twelve restaurants that are nearby. The weather is getting bad or they’re not interest in any of those restaurants at the moment, then, they can choose to stay home and search for any name and the application will display all the movies that contains that searched name for them to choose.

Languages: HTML, CSS and JavaScript.
APIs: Weather API, TripAdvisor (Restaurants), and IMDb (Movies).
Tools: Visual Studio Code.

Everybody is busy in today's world, so tracking your everyday tasks is a crucial tool in productivity. With the Task Board, we have made it easy for users to manage their tasks. Users can see all tasks and find specific tasks with ease with a filter/search feature. The User can also create new tasks, along with updating and deleting current tasks. Users can access this application via an interactive single paged application hosted and deployed using a CI/CD pipeline to an AWS S3 bucket that interacts with a RESTful API hosted on an AWS EC2.

Languages: Front-End: HTML, CSS, Bootstrap, TypeScript, Angular (Single Page Application).
APIs: RESTful API hosted on an AWS EC2.
Tools: Visual Studio Code.

The Banking API will manage the bank accounts of its users. It will be managed by the Bank's employees and admins. Employees and Admins count as Standard users with additional abilities. All users must be able to update their personal information, such as username, password, first and last names, as well as email. Accounts owned by users must support withdrawal, deposit, and transfer. Transfer of funds should be allowed between accounts owned by the same user, as well as between accounts owned by different users. Standard users should be able to register and login to see their account information. They can have either Checking or Savings accounts. Employees can view all customer information, but not modify in any way. Admins can both view all user information, as well as directly modify it.

Languages: Java, JDBC, Servlet, MySQL.
APIs: Created API and Endpoints.
Tools: Spring Tool Suite 4, Postman, DBeaver.
